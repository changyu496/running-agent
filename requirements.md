# 跑步Agent - 产品需求文档

## 1. 项目概述

### 1.1 产品定位
一个基于AI的跑步数据分析与指导工具，帮助用户记录、分析和改进跑步表现。

### 1.2 核心功能
1. **备忘录分析**：通过文本备忘录 + 高驰数据截图，AI评价跑步情况
2. **视频跑姿分析**：上传跑步视频，AI分析跑姿并提供改进建议
3. **历史数据积累**：自动保存所有分析记录，支持关联和统计
4. **智能指导**：基于历史数据提供个性化跑步建议

### 1.3 技术栈
- **前端**：Electron + React（Mac桌面应用）
- **后端**：Python + FastAPI
- **AI模型**：Qwen（视觉分析）+ MediaPipe（姿态估计）
- **存储**：SQLite + 本地文件系统
- **配置**：JSON配置文件

## 2. 功能需求

### 2.1 备忘录分析模块

#### 2.1.1 输入
- **文本输入**：纯文本备忘录（用户手动输入）
- **图片上传**：高驰App数据截图（PNG/JPG格式）

#### 2.1.2 处理流程
1. 用户输入备忘录文本
2. 用户上传高驰截图
3. 点击"开始分析"按钮
4. 调用Qwen视觉模型分析截图内容
5. 结合备忘录文本生成综合评价
6. 显示分析结果
7. 用户点击"保存记录"保存到数据库

#### 2.1.3 输出内容
- **数据概览**：距离、配速、心率、步频等关键指标
- **表现评价**：整体评价、优点、需要改进的地方
- **改进建议**：基于本次数据和历史数据的针对性建议
- **历史对比**：与历史同距离/同配速的对比

#### 2.1.4 错误处理
- 图片识别失败：提供重试按钮
- 支持备选模型（如Claude/GPT-4V）作为备选
- 用户可手动纠正数据后重新分析

### 2.2 视频跑姿分析模块

#### 2.2.1 输入
- **视频上传**：跑步视频（MP4/MOV格式，建议10-30秒）
- **拍摄角度选择**：侧面 / 背面（用户需在上传时选择）

#### 2.2.2 处理流程
1. 用户上传跑步视频
2. **选择拍摄角度**：侧面 或 背面
3. 点击"开始跑姿分析"按钮
4. **本地处理**：使用MediaPipe Pose提取关键点数据
   - 提取33个身体关键点
   - 根据拍摄角度计算不同的关键指标
   - 生成骨骼可视化图（类似Ochy效果）
5. **AI分析**：将关键点数据、可视化图和拍摄角度信息发送给Qwen分析
6. 显示分析结果（可视化图 + 数值指标 + AI评价）
7. 用户点击"保存记录"保存到数据库

#### 2.2.3 输出内容（根据拍摄角度区分）

**侧面拍摄分析重点：**
- **可视化展示**：侧面骨骼关键点连线图 + 角度标注
- **关键角度分析**：
  - 头部倾斜角度
  - 躯干前倾角度
  - 膝关节角度（支撑腿和摆动腿）
  - 踝关节角度
  - 手臂摆动角度
- **步态分析**：
  - 步幅
  - 落地方式（前脚掌/中足/后脚跟）
  - 触地时间
  - 垂直振幅
- **跑姿评估**：整体评分、优点、需要改进的地方
- **针对性建议**：具体的训练建议和练习方法

**背面拍摄分析重点：**
- **可视化展示**：背面骨骼关键点连线图 + 角度标注
- **对称性分析**：
  - 左右平衡（左右脚落地时间对比）
  - 骨盆倾斜角度
  - 肩膀高度对称性
  - 手臂摆动对称性
- **对齐度分析**：
  - 身体中心线对齐
  - 脚落地位置（内翻/外翻）
  - 膝盖对齐（内扣/外翻）
- **跑姿评估**：整体评分、对称性问题、需要改进的地方
- **针对性建议**：针对不对称问题的训练建议

#### 2.2.4 技术方案
- **姿态估计**：MediaPipe Pose（轻量、快速、Mac兼容性好）
- **关键点提取**：33个身体关键点（肩膀、肘部、手腕、髋部、膝盖、脚踝等）
- **角度计算**：基于关键点计算身体各部位角度
  - **侧面**：计算头部倾斜、躯干前倾、膝关节角度、踝关节角度等
  - **背面**：计算左右平衡、骨盆倾斜、肩膀对称性、身体对齐度等
- **对称性分析**（仅背面）：计算左右脚落地时间差、左右侧关键点对称性
- **可视化**：使用OpenCV或PIL绘制骨骼图和角度标注

#### 2.2.5 错误处理
- 视频分析失败：提示用户，非必须功能，不影响其他功能使用

### 2.3 历史记录模块

#### 2.3.1 记录列表
- 显示所有历史记录（按时间倒序）
- 每条记录显示：日期、类型（备忘录/视频）、关键指标（距离、配速等）
- 支持搜索和筛选（按类型、日期范围）

#### 2.3.2 记录详情
- 点击记录跳转到详情页
- 详情页显示完整信息：
  - 备忘录内容（如有）
  - 高驰数据（如有）
  - 视频分析结果（如有）
  - AI分析评价
  - 关联的其他记录（如有）
- 支持返回列表（便捷导航）

#### 2.3.3 记录关联
- **自动关联**：AI优先判断是否同一次跑步
  - 判断标准：时间间隔 < 30分钟，且距离/配速相似
- **手动关联**：用户可手动关联/解除关联多条记录
- **关联展示**：关联的记录在详情页中显示

### 2.4 数据统计模块

#### 2.4.1 统计维度
- **总览指标**：总跑步次数、总距离、平均配速、平均心率
- **趋势分析**：
  - 配速趋势（时间轴图表）
  - 心率趋势（时间轴图表）
  - 距离分布（饼图/柱状图）
  - 步频趋势
  - 步幅趋势
  - 训练强度分布（有氧/无氧）
  - 跑步时间分布（早晨/下午/晚上）

#### 2.4.2 可视化
- 使用图表库（如Chart.js/ECharts）展示趋势
- 支持时间范围筛选（最近一周/一月/三月/全部）

### 2.5 智能指导模块

#### 2.5.1 触发方式
- 用户主动点击"智能指导"页面查看

#### 2.5.2 内容生成
- **本周训练总结**：基于本周数据生成总结
- **趋势分析**：分析最近一个月的进步/下降趋势
- **下周训练建议**：基于当前水平和趋势，提供具体训练计划
- **注意事项**：健康提醒、训练强度建议等

#### 2.5.3 数据来源
- 基于所有历史记录数据
- 结合用户基础信息（年龄、身高、体重等）

## 5. 技术实现细节

### 5.1 视频分析技术方案

#### 5.1.1 MediaPipe Pose
```python
import mediapipe as mp
import cv2

# 初始化MediaPipe Pose
mp_pose = mp.solutions.pose
pose = mp_pose.Pose(
    static_image_mode=False,
    model_complexity=1,
    enable_segmentation=False,
    min_detection_confidence=0.5
)

# 处理视频帧
cap = cv2.VideoCapture(video_path)
while cap.isOpened():
    ret, frame = cap.read()
    if not ret:
        break
    
    # 提取关键点
    results = pose.process(cv2.cvtColor(frame, cv2.COLOR_BGR2RGB))
    
    # 获取关键点坐标
    if results.pose_landmarks:
        landmarks = results.pose_landmarks.landmark
        # 计算角度、绘制骨骼图等
```

#### 5.1.2 关键点数据结构
- MediaPipe提供33个关键点
- 关键点包括：头部、肩膀、肘部、手腕、髋部、膝盖、脚踝等
- 每个关键点包含：x, y, z坐标和可见性

#### 5.1.3 角度计算
- **头部倾斜**：基于头部和肩膀关键点
- **躯干前倾**：基于肩膀和髋部关键点
- **膝关节角度**：基于髋部、膝盖、脚踝关键点
- **踝关节角度**：基于膝盖、脚踝、脚尖关键点

#### 5.1.4 可视化
- 使用OpenCV绘制骨骼连线
- 标注关键角度（橙色框）
- 生成静态图片或GIF动画

## 8. 配置文件需求清单

详细的个人基本信息需求清单请参考：`personal-info-checklist.md`

### 8.1 用户基础信息（必填）
- **姓名**：用于记录标识、个性化称呼
- **性别**（male/female）：用于心率区间计算、训练建议参考
- **年龄**：用于计算最大心率（220 - 年龄）、训练强度评估
- **身高**（cm）：用于步幅分析、BMI计算、训练建议
- **体重**（kg）：用于BMI计算、训练强度评估、卡路里消耗估算
- **最大心率**（bpm）：用于心率区间划分、训练强度评估（如不确定可留空，系统自动计算）
- **静息心率**（bpm）：用于心率储备计算、有氧能力评估

### 8.2 训练目标（可选，用于更精准的分析）
- **目标配速**：用于训练建议、进步评估
- **周跑量目标**：用于训练计划制定
- **周训练次数**：用于训练计划制定
- **跑龄**：用于训练建议的难度调整
- **当前水平**：初学者/中级/高级
- **主要训练目标**：减脂/提升速度/提升耐力/保持健康

### 8.2.1 个人最佳成绩（可选，强烈推荐）
- **半马最佳成绩**：用于评估当前水平、制定训练计划、配速区间计算
- **全马最佳成绩**：用于评估耐力水平、制定长距离训练计划
- **10公里最佳成绩**：用于评估速度能力、制定速度训练计划
- **用途**：
  - 自动评估跑者水平（根据成绩自动判断：初学者/中级/高级）
  - 计算个性化配速区间（轻松跑、有氧跑、节奏跑、间歇跑等）
  - 制定针对性训练建议
  - 对比分析进步情况

### 8.3 健康信息（可选）
- **是否有伤病**：用于训练建议、注意事项提醒
- **伤病详情**：如有伤病，请说明具体情况

### 8.4 API配置（必填）
- **Qwen API Key**：调用Qwen模型进行AI分析
- **Qwen API URL**：API调用地址
- **备选模型配置**（可选）：当Qwen识别失败时的备选方案（Claude/GPT-4V等）

### 8.5 应用设置（可选）
- **主题**（light/dark）：界面主题，默认浅色
- **自动关联阈值**（分钟）：AI判断"同一次跑步"的时间间隔，默认30分钟
