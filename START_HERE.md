# 跑步Agent - 启动说明

## 一键启动（推荐）

在项目根目录执行：

```bash
./start.sh
```

脚本会自动：
1. 启动后端（若未运行）
2. 启动前端（若未运行）
3. 打开 **跑步Agent** 的 Mac 应用窗口（Electron）

**请使用弹出的「跑步Agent」窗口操作，不要用浏览器打开 localhost。**

关闭窗口后，后端与前端仍在运行；再次执行 `./start.sh` 可重新打开应用。

---

## 视频跑姿分析

- **可视化视频需 ffmpeg**：为让骨架+角度叠加视频能在浏览器正常播放，请安装 ffmpeg：`brew install ffmpeg`。未安装时仍会生成视频，但可能无法在应用内播放。
- **画面倒置**：若视频上下颠倒，勾选「画面倒置时勾选（旋转 180°）」后重新分析。需等待分析完成（1–3 分钟）才会看到新结果。
- 若视频无法播放：会显示「可视化视频加载失败」并回退到原视频预览；可重新分析一次（安装 ffmpeg 后）或检查后端日志。

## 日志在哪里看？

- **后端日志**：执行 `./start.sh` 时，后端在后台运行，日志会输出到**同一终端**。分析视频时留意 `[姿态分析]`、`[视频分析]` 等前缀。
- **单独看后端**：另开终端执行 `cd backend && python3 main.py`，所有日志会直接显示。

## 分析没反应时

- 若页面顶部出现 **「后端未连接」** 红条：说明后端未启动，请先执行 `./start.sh` 或单独运行 `cd backend && python3 main.py`。
- 若点击「开始分析」后一直显示「分析中」：可能是网络或 Qwen API 问题，请查看运行后端的终端里是否有报错。

---

## 手动启动（可选）

若需要分别调试：

```bash
# 终端1 - 后端
cd backend && python3 main.py

# 终端2 - 前端 + 应用窗口
cd frontend && npm run dev
```

同样请使用最后弹出的 **跑步Agent** 窗口，不要用浏览器。
